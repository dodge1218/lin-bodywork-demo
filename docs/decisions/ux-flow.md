# UX Flow

1. User lands on `/` and sees value prop + primary CTA above the fold.
2. User scans services and results proof blocks.
3. User can validate fit via `/services`, `/about`, `/results` pages.
4. Every page includes CTA route to `/get-started`.
5. Conversion step: complete strategy-call form on `/get-started` (or `/contact`).

Notes:
- CTA repeated in header, hero, and bottom conversion block.
- Mobile-first spacing and tap-friendly controls used throughout.
